## 📖 Project Overview
This repository contains the infrastructure as code (IaC) and configuration files for a complete **Observability and Performance Evaluation System** deployed on Kubernetes. 

The project demonstrates how to provision a lightweight K3s cluster (with seamless migration to AWS EKS) and integrate the **LGTM Stack (Loki, Grafana, Tempo, Mimir)** to monitor system health, collect logs, and analyze metrics under heavy load generated by **k6**.

## 🏗️ Architecture & Tech Stack

* **Infrastructure:** K3s (Local/VM environment) & AWS EKS (Production environment).
* **Log Aggregation:** Grafana Loki + Promtail (Log forwarding).
* **Metrics Storage:** Grafana Mimir (Scalable Prometheus alternative).
* **Visualization:** Grafana Dashboards.
* **Load Testing:** Grafana k6 (Baseline, Stress, and Spike testing).

## 🚀 Key Features
1.  **Automated Telemetry:** Real-time collection of node and pod-level metrics using kube-state-metrics and node-exporter.
2.  **Centralized Logging:** Aggregating container logs using Promtail and storing them efficiently in Loki.
3.  **Performance Benchmarking:** Custom k6 scripts designed to evaluate system saturation points and calculate RPS/P90 metrics.
4.  **Infrastructure as Code:** Declarative Kubernetes manifests for reproducible deployments.

## 📂 Repository Structure
```text
├── manifests/            # Kubernetes YAML configurations
│   ├── lgtm-stack/       # Loki, Grafana, Mimir deployments
│   ├── monitoring/       # Promtail, Exporters, ServiceMonitors
│   └── apps/             # Target applications for load testing
├── dashboards/           # Exported Grafana Dashboard JSON models
├── k6-scripts/           # Load testing scenarios
│   ├── baseline.js       # Normal traffic simulation
│   ├── stress_test.js    # System limit discovery
│   └── spike_test.js     # Sudden traffic surge simulation
└── docs/                 # Architecture diagrams and performance reports
```

⚙️ Quick Start (Local Deployment)
1. Clone the repository:
```Bash
git clone [https://github.com/hmdat-1706/k8s-observability-lgtm.git](https://github.com/hmdat-1706/k8s-observability-lgtm.git)
cd k8s-observability-lgtm
```

2. Deploy the LGTM Stack:
```Bash
kubectl apply -k manifests/lgtm-stack/
kubectl apply -k manifests/monitoring/
```

3. Verify Pod Status:
```Bash
kubectl get pods -n observability
```

4. Access Grafana:
```Bash
kubectl port-forward svc/grafana 3000:80 -n observability
# Access at http://localhost:3000
```

📊 Performance Testing Execution
To run the performance evaluation scripts against the deployed cluster, ensure k6 is installed on your local machine:
```Bash
# Run a baseline test to establish normal metrics
k6 run k6-scripts/baseline.js
# Run a spike test to observe system recovery via Mimir & Loki
k6 run k6-scripts/spike_test.js
```
